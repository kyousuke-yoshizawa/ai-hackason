# GitHub 操作スキル

このスキルは GitHub MCP ツールを使用して、PR作成、Issue作成、ブランチ管理、ファイル操作など、GitHub上の様々な操作を対話的にサポートします。

## 📋 絶対ルール

1. **日本語記載**：すべての PR タイトル、説明、コミットメッセージは日本語で記載します
2. **ブランチ戦略**：GitHub 操作時は必ず新しいブランチを作成してから実行します
3. **操作者表示**：ブランチ名に操作者を含めて、誰が何をしたのか明確にします
4. **簡潔丁寧**：PR とレポートは簡潔で分かりやすく、丁寧に記載します

## 実行手順

このスキルが呼び出されたら、以下の手順に従ってください：

### ステップ 0: 操作者の確認（必須）
**必ず最初に**ユーザー/チームメンバーが誰なのかを確認します。

**質問内容**：「どなたが作業されていますか？」
**選択肢（4択）**：
- 吉沢（yoshizawa）
- 板垣（itagaki）
- 佐藤（sato）
- 高柳（takayanagi）

### ステップ 1: ブランチ確認と使用（固定ルール・変更不可）
このプロジェクトでは各メンバー専用の固定ブランチを使用します。

**許可されるブランチは以下の 5 つだけ**：
- `main` — 本番ブランチ（PR マージ先のみ）
- `feature/yoshizawa` — 吉沢専用
- `feature/itagaki` — 板垣専用
- `feature/sato` — 佐藤専用
- `feature/takayanagi` — 高柳専用

**重要な制約**：
- **他のブランチ名は絶対に作らない** （例：`feature/login-page-yoshizawa` はNG、`feature/fix-bug` はNG）
- **ステップ0で確認したメンバーに対応するブランチを使用**
- メンバーのブランチが既に存在していれば、そこに新しい commit を重ねる
- ブランチは全期間（20時間ハッカソン）存在し、複数の PR に使われる

### ステップ 2: 操作カテゴリの選択
確認したメンバーに対応するブランチを使用することを宣言し、AskUserQuestion ツールで操作カテゴリを質問してください。

例：「吉沢さんの作業ですね。`feature/yoshizawa` ブランチで進めます」

**質問項目**：
- header: "実行する操作"
- 選択肢1: "PR操作" - PR作成、更新、マージ、確認
- 選択肢2: "Issue操作" - Issue作成、検索、確認
- 選択肢3: "ブランチ操作" - ブランチ作成、一覧表示
- 選択肢4: "ファイル操作" - ファイル作成・更新・削除・確認
- 選択肢5: "コミット確認" - リポジトリのコミット一覧表示

### ステップ 3: 詳細情報の取得
ユーザーの選択した操作カテゴリに応じて、必要な詳細情報を段階的に AskUserQuestion で質問してください。

#### PR操作の場合：
- PR作成：base ブランチ、head ブランチ、タイトル、説明
- PR更新：PR番号、更新内容（タイトル/説明/ドラフト状態など）
- PR マージ：PR番号、マージ方法（merge/squash/rebase）
- PR確認：PR番号

#### Issue操作の場合：
- Issue作成：タイトル、説明、ラベル（オプション）
- Issue検索：検索キーワード、フィルタ条件（状態/ラベルなど）
- Issue確認：Issue番号

#### ブランチ操作の場合：
- ブランチ作成：新ブランチ名、基点ブランチ
- ブランチ一覧：（パラメータなし）

#### ファイル操作の場合：
- ファイル作成・更新：ファイルパス、ファイル内容、コミットメッセージ
- ファイル削除：ファイルパス、コミットメッセージ
- ファイル確認：ファイルパス

#### コミット確認の場合：
- （パラメータなし、直近コミット一覧を表示）

### ステップ 4: GitHub MCP ツールの実行
ユーザーが提供した情報に基づいて、以下の手順で GitHub MCP ツールを実行してください：

1. **リポジトリ情報の取得**：
   - 現在の git リポジトリから owner と repo を取得
   - デフォルト：`kyousuke-yoshizawa/ai-hackason`

2. **ToolSearch でスキーマ取得**：
   - 使用するツール（例：`mcp__github__create_pull_request`）のスキーマを ToolSearch で取得

3. **ツール実行**：
   - 取得したスキーマに基づいて、ユーザーの情報を入力してツール実行

### ステップ 5: 結果の報告
GitHub MCP ツール実行後、以下の情報をユーザーに報告してください：
- 操作の成功/失敗
- 作成または更新されたオブジェクトの詳細（URL、IDなど）
- 次に実行可能な操作の提案

## 使用方法

```
/github-operations
```

実行すると、以下のステップで操作を進めます：

1. **操作カテゴリを選択**：PR/Issue/ブランチ・ファイル操作から選択
2. **詳細情報を入力**：選択した操作に必要な情報を段階的に指定
3. **GitHub MCP ツールを実行**：選択した内容に応じて適切なツールを実行
4. **結果を報告**：操作結果をユーザーに表示

## GitHub CLI（`gh`）を使った代替手順

このスキルの代わりに、GitHub CLI を使用して PR 作成・マージを行うこともできます：

```bash
# PR 作成
gh pr create --title "タイトル" --body "説明"

# PR マージ
gh pr merge <PR番号> --merge
```

詳細は `docs/ai-hackathon-team-ops/references/pr-review-merge-flow.md` の「2-B. CLI（`gh`）を使った PR 作成・マージ」を参照してください。

**gh CLI が必要な場合**:
- MCP ツールが利用できない場合
- より細かい操作制御が必要な場合
- 自動化スクリプトを書く場合

## 対応する操作

### PR操作
- **PR作成**：新しいプルリクエストを作成
- **PR更新**：既存のプルリクエストを更新
- **PR マージ**：プルリクエストをマージ
- **PR確認**：プルリクエスト情報を取得・確認

### Issue操作
- **Issue作成**：新しいイシューを作成
- **Issue検索**：特定の条件でイシューを検索
- **Issue読取**：イシュー情報を取得・確認

### ブランチ・コミット操作
- **ブランチ作成**：新しいブランチを作成
- **ブランチ一覧**：リポジトリのブランチ一覧を表示
- **コミット確認**：リポジトリのコミット情報を表示

### ファイル操作
- **ファイル作成・更新**：ファイルを作成または更新
- **ファイル削除**：ファイルを削除
- **ファイル内容取得**：ファイルの内容を取得

## リポジトリの自動判定

このスキルは実行時に現在の git リポジトリから owner と repo 名を自動判定します。デフォルトでは `kyousuke-yoshizawa/ai-hackason` をターゲットします。

## 重要な注意事項

### ブランチ戦略の必須性（固定ルール）

このプロジェクトでは、固定された 4 つのメンバーブランチを使用します（`main` を除く）。

- **各メンバーは専用ブランチを使用** — 吉沢は `feature/yoshizawa`、板垣は `feature/itagaki`、佐藤は `feature/sato`、高柳は `feature/takayanagi`
- **新しいブランチを作成しない** — 上記 4 つ以外のブランチは絶対に作らない
- **メンバーのブランチ内で複数の commit を重ねる** — 各メンバーが 1 つのブランチで複数の PR を出す可能性もある
- **操作者の履歴が明確に残る** — ブランチ名でメンバーが確定
- **複数メンバーが同時に作業できる** — 4 つの独立したブランチで並行作業

### コミットメッセージとPR説明の言語

- **すべて日本語で記載**
- 簡潔で分かりやすく
- 丁寧な表現を心がける
- 技術的な詳細も必要に応じて含める

### 例

**良いコミットメッセージ**：
```
ログイン画面の基本構造を実装

- React コンポーネント作成
- Tailwind CSS でスタイリング
- フォーム入力の基本実装
```

**良い PR 説明**：
```
## 概要
ログイン画面のスケルトンを作成しました。

## 変更内容
- LoginPage コンポーネントの実装
- 基本的なフォームレイアウト
- Tailwind CSS による初期スタイル

## 次のステップ
- バックエンド認証の統合
- フォーム検証の追加
```
