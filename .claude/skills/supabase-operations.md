# Supabase 操作スキル

このスキルは Supabase データベースの操作を対話的にサポートします。テーブル管理とデータ管理の両方に対応しています。

## 対応機能

### 📋 テーブル管理
- **テーブル作成**：新しいテーブルを定義して作成
- **テーブル更新**：カラムの追加・削除・型変更
- **テーブル削除**：不要なテーブルを削除
- **テーブル閲覧**：テーブル一覧とスキーマ確認

### 📊 データ管理
- **データ追加**：新しいレコードを挿入
- **データ更新**：既存レコードを修正
- **データ削除**：レコードを削除
- **データ閲覧**：クエリでデータを検索・表示

## 使用方法

```
/supabase-operations
```

実行すると、対話的に操作を進めます。

## 実行手順

このスキルが呼び出されたら、以下の手順に従ってください：

### ステップ 1: 操作カテゴリの選択
AskUserQuestion で操作カテゴリを質問してください：

**質問内容**：「何を操作しますか？」
**選択肢**：
- テーブル管理
- データ管理
- スキーマ確認

### ステップ 2: 具体的な操作の選択
選択したカテゴリに応じて詳細な操作を質問：

#### テーブル管理の場合
- テーブル作成
- テーブル更新
- テーブル削除
- テーブル一覧閲覧

#### データ管理の場合
- データ追加（単一/複数）
- データ更新（条件指定）
- データ削除（条件指定）
- データ閲覧（フィルタ・ソート・ページング）

#### スキーマ確認の場合
- テーブルスキーマ表示
- カラム情報確認
- インデックス・制約確認

### ステップ 3: 詳細情報の取得
操作に必要な情報を段階的に質問：

#### テーブル作成の場合
- テーブル名
- カラム定義（名前、型、制約）
- プライマリキー設定

#### データ追加の場合
- 対象テーブル
- カラム値の入力
- 複数レコード追加オプション

#### データ更新の場合
- 対象テーブル
- 更新条件（WHERE句）
- 更新内容

#### データ削除の場合
- 対象テーブル
- 削除条件（WHERE句）
- 確認メッセージ

#### データ閲覧の場合
- 対象テーブル
- フィルタ条件（オプション）
- ソート順（オプション）
- 表示件数

### ステップ 4: SQL クエリの生成と実行
入力された情報に基づいて：

1. **適切な SQL クエリを生成**
   ```sql
   CREATE TABLE users (
     id SERIAL PRIMARY KEY,
     name VARCHAR(255) NOT NULL,
     email VARCHAR(255) UNIQUE
   );
   ```

2. **環境確認**
   - Supabase 接続情報の確認
   - プロジェクト URL とキーの確認

3. **クエリ実行**
   - Supabase API を使用してクエリを実行
   - または SQL エディタの使用手順を案内

### ステップ 5: 結果の報告
実行結果を以下の形式で報告：

```
【操作結果】
ステータス：✅ 成功 / ❌ 失敗

【詳細】
- 実行内容：[操作内容]
- 影響行数：[XX 件]
- 実行時間：[XX ms]

【確認すべき項目】
- 期待通りの結果になったか
- データの整合性は取れているか
```

## Supabase 接続情報の確認方法

スキル実行時は、以下の環境変数が設定されていることを確認：

```
SUPABASE_URL=https://[project].supabase.co
SUPABASE_KEY=[anon/service_role キー]
```

※ プロジェクト URL とキーは `.env` ファイルまたは環境変数で指定

## データ型対応表

| 論理型 | SQL型 | 説明 |
|--------|-------|------|
| 数値 | INT, BIGINT, NUMERIC | 整数・小数 |
| 文字列 | VARCHAR, TEXT | 短・長文字列 |
| 日時 | TIMESTAMP, DATE | 日付・時刻 |
| 真偽 | BOOLEAN | TRUE/FALSE |
| JSON | JSONB | 構造化データ |
| UUID | UUID | 一意識別子 |

## 重要な注意事項

### セキュリティ
- 機密情報（API キーなど）を表示しない
- SQL インジェクション対策：パラメータ化クエリを使用
- 削除操作は慎重に、確認メッセージを表示

### ベストプラクティス
- 大量削除は WHERE 句で条件を指定
- バックアップが必要な場合は事前に確認
- 本番環境への変更は慎重に

## 使用例

### テーブル作成の例
```
【操作】テーブル作成
【テーブル名】products
【カラム定義】
  - id: INT PRIMARY KEY
  - name: VARCHAR(255)
  - price: NUMERIC
  - created_at: TIMESTAMP DEFAULT NOW()
```

### データ追加の例
```
【操作】データ追加
【テーブル】products
【データ】
  - name: "新商品A"
  - price: 1000
```

### データ閲覧の例
```
【操作】データ閲覧
【テーブル】products
【条件】price > 500
【ソート】created_at DESC
【件数】10 件
```
