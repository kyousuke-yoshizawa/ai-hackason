工程 ① 設計・インフラ

設計からインフラまで、AIに深く関与させる

設計を残す

対話で仕様を固め、CLAUDE.md（18KBの規約）とarchitecture-auditの手順書にT番号タスクとして記録。

構築を任せる

infra-builder サブエージェントが Vercel / Supabase / GitHub Actions / cron を構築。

自動化する

Supabase×Vercel統合で環境変数を自動注入、main pushで自動デプロイ、npm run setup がシークレット自動生成。

🛠 実インフラ判断もAIと：Vercel Hobbyの「cron 1日1回」制限に直面 → 30分毎の混雑通知を GitHub Actions schedule へ移行。

工程 ② 実装

「AIに書かせる」の一歩先へ

実装→レビューの自動ループ
frontend / backend / infra → reviewer が採点
REVIEW: ISSUES → 修正 ⟳ REVIEW: CLEAN で完了

実装エージェントの自己申告だけでは完了としない。CLEANになるまで反復する品質ゲート。

AIをレールに乗せる

CLAUDE.md でブランチ戦略（5本のみ・1人1本）、secretsハードコード禁止、strict TS（no any）、パターンを統一。

成果：8画面・16コンポーネント・複数ドメインを実装。Claude API統合も claudeClient / promptBuilder / scoring / schema に分割。

AIレビューをゲートに

reviewer サブエージェントが変更を採点し、指摘があれば修正ループへ。人の目に頼りきらない。

実環境でQA

PRごとに Vercelプレビュー が自動で立ち上がり、マージ前に本番同等の環境で動作確認。
