# プラン生成のClaude API概算コスト（Issue #149）

要件定義書v2 8章「⚠️ Claude API利用料：チーム共有キーで管理」に対応する概算コスト試算。実測値は `POST /api/plan/generate` が出力する構造化ログ（`evt: 'plan_generated'`）から確認できる（`backend/domains/plan/claudeClient.ts` / `api/plan/generate.ts`）。

## 単価（2026-07-16、docs.claude.com確認済み）

Claude Sonnet 5（2026年8月31日までの導入価格）:

| 項目 | 単価 |
|---|---|
| Input tokens | $2 / MTok（100万トークンあたり） |
| Output tokens | $10 / MTok（100万トークンあたり） |

> 2026年9月1日以降は $3 / $10 → $3 / $15 の標準価格に変わる予定（docs.claude.com記載）。本ハッカソンのデモ期間（〜7/25）は導入価格が適用される。

## 計算式

```
コスト（USD） = (input_tokens / 1,000,000 × $2) + (output_tokens / 1,000,000 × $10)
```

## 1回あたりの概算（実測前の見積もり）

店舗8件のコンテキスト＋意図解析＋複数案生成のプロンプト構成から、以下を暫定的な目安とする（**実測値は #149 のログ・#148 のレイテンシ計測結果で置き換えること**）：

| 項目 | 目安 |
|---|---|
| Input tokens | 約1,500〜2,500 |
| Output tokens | 約800〜1,500 |
| 1回あたりコスト | 約 $0.011〜$0.020 |

## デモ100回想定での概算

| ケース | Input | Output | 概算コスト |
|---|---|---|---|
| 少なめ（1,500 / 800） | 150,000 tokens | 80,000 tokens | $0.30 + $0.80 = **$1.10** |
| 多め（2,500 / 1,500） | 250,000 tokens | 150,000 tokens | $0.50 + $1.50 = **$2.00** |

デモ期間・練習も含め100回程度の呼び出しでは、いずれのケースでも**数ドル程度**に収まる見込み。#150のレート制限（10回/分）と合わせ、コスト暴走のリスクは低い。

## 実測値の反映方法

`api/plan/generate.ts` が成功時に出力するログ（Vercelのログから確認可能）:

```json
{"evt":"plan_generated","ms":<レイテンシms>,"input_tokens":<入力トークン数>,"output_tokens":<出力トークン数>,"model":"claude-sonnet-5","candidates":<案数>}
```

実測後、本ドキュメントの「1回あたりの概算」を実測値に更新すること。
