# マスタ画面 検索・絞り込み・ソート機能 設計書

**日付**: 2026-07-15
**対象画面**: 店舗マスタ（`StoreManagementPanel`）、ユーザーマスタ（`UserManagementPanel`）、店舗一覧（`StoresPage`）

## 背景・目的

管理者向けマスタ画面（店舗管理・ユーザー管理）および一般ユーザー向け店舗一覧に、検索・絞り込み・ソート機能がなく、件数が増えると目的の行を探しにくい。3画面に対して検索条件の指定・部分一致検索・並び替えを追加する。

## 方針

3画面とも「取得済み一覧データをクライアント側でリアルタイムに絞り込み・ソートする」方式に統一する。API呼び出しは初回の一覧取得のみとし、検索・フィルタ・ソートはブラウザ側の `useMemo` で完結させる。既存の `src/pages/LikesListPage.tsx` にあるクライアントサイド絞込・ソートの実装パターン（`sortKey` の `<select>` + `useMemo`）を踏襲する。

検索は「検索ボタン」を設けず、テキスト入力・プルダウン選択のたびにリアルタイムに反映する（クライアント側処理のため、ボタンによる明示的な発火は不要と判断）。

## 画面ごとの仕様

### 店舗マスタ（`src/components/StoreManagementPanel.tsx`）

- **検索**: 店舗名のテキスト入力（部分一致・大文字小文字を区別しない）
- **絞り込み**: カテゴリのプルダウン（取得済み一覧データから動的に選択肢を生成）
- **ソート**: テーブルヘッダー（名前・カテゴリ・営業開始時間）をクリックしてソート。昇順/降順をトグルし、▲▼アイコンで現在の状態を表示
- **変更点**: 既存の `categoryFilter` によるサーバー完全一致クエリ（`GET /api/stores?category=...`）を撤去し、無条件で一覧を取得した上でクライアント側の部分一致検索に置き換える

### ユーザーマスタ（`src/components/UserManagementPanel.tsx`）

- **検索**: 名前・メールのテキスト入力（部分一致、いずれかにマッチすれば表示）
- **絞り込み**: ロールのプルダウン（admin/store_manager/user）、有効/無効のプルダウン
- **ソート**: テーブルヘッダー（メール・名前・ロール）をクリックしてソート
- **付随修正**: `loadUsers` が `GET /api/users` を無条件呼び出ししており、サーバー側デフォルトの `limit=20`（`server/routes/users.ts` 39行目）により21件目以降が取得できていない。検索・絞り込みが全件を対象としないと機能として意味を持たないため、`GET /api/users?limit=100`（サーバー許容の最大値）を明示指定するよう修正する

### 店舗一覧（`src/pages/StoresPage.tsx`）

- **検索**: 店舗名のテキスト入力
- **絞り込み**: カテゴリのプルダウン
- **ソート**: `LikesListPage.tsx` と同様のプルダウン形式（店舗名順・カテゴリ順）を採用。カード一覧のためテーブルヘッダークリックは使わない

## 共有部品

テーブルヘッダーのソートボタン（クリックで昇順⇄降順、▲▼表示）は店舗マスタ・ユーザーマスタで見た目・挙動が同一のため、小さな共通コンポーネント `SortableHeader` として切り出し、両画面で再利用する。

- Props: `label`（表示名）、`sortKey`（このヘッダーが対応するキー）、`currentSortKey`、`currentSortDir`、`onSort`（クリック時コールバック）
- 店舗一覧（StoresPage）はプルダウン方式のため対象外

## 空状態・エラー表示

一覧が0件の場合の文言を、状況に応じて出し分ける。

- 元データ自体が0件: 既存の「店舗がありません」「ユーザがいません」を維持
- 検索・絞り込みの結果0件: 「検索条件に一致する〜がありません」と表示し、データ自体はあるが条件に一致しないことを明示する

## スコープ外

- サーバー側でのページング・検索クエリ対応（今回はクライアント側処理に統一するため対象外）
- ユーザーマスタの101件目以降への対応（`limit=100` で十分とみなす。ハッカソン規模のテストデータでは超過しない想定）
- 店舗一覧（StoresPage）へのテーブルヘッダー方式の適用（カード一覧のため対象外）

## テスト方針

- 各画面のフィルタ・検索・ソートに対する単体テスト（`tests/unit/`）を追加
- 既存のカテゴリ完全一致フィルタに関するテストがあれば、部分一致への変更に合わせて更新
- ユーザーマスタの `limit=100` 指定について、21件以上のユーザーが一覧に表示されることを確認するテストを追加
