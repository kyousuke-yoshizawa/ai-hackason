# Notion 実装ログデータベステンプレート

## データベース基本情報

| 項目 | 値 |
|------|-----|
| **データベース名** | 実装ログ |
| **説明** | GitHub PR マージ時に自動記録される実装内容のログ |
| **公開範囲** | チーム全体で閲覧可能 |

## 必須カラム（自動入力）

GitHub Actions により自動で入力されるカラムです。

### 1. PRタイトル（Title）

- **型**：Title
- **説明**：PR のタイトル
- **入力方法**：自動（GitHub から）
- **例**：「ログイン機能とダッシュボード画面を実装」

### 2. 担当者（Text）

- **型**：Text
- **説明**：実装者の GitHub ユーザ名
- **入力方法**：自動（GitHub から）
- **例**：「yoshizawa」

### 3. PRリンク（URL）

- **型**：URL
- **説明**：GitHub PR への直接リンク
- **入力方法**：自動（GitHub から）
- **例**：「https://github.com/dx-ai-sas-com-organization/repository-007/pull/7」

## 推奨カラム（手動入力）

実装完了後に手動で入力するカラムです。

### 4. 実装日時（Date）

- **型**：Date
- **説明**：PR マージ日時
- **入力方法**：手動
- **推奨**：自動入力ルール「自動入力 → 現在の日付」

### 5. ステータス（Select）

- **型**：Select
- **説明**：実装の進捗状況
- **選択肢**：
  - 🔴 **未確認**：実装完了だが確認待ち
  - 🟡 **確認中**：テスト・確認実施中
  - 🟢 **完了**：確認完了・本番反映済み
- **デフォルト**：「未確認」
- **入力方法**：手動

### 6. 機能分類（Select / Multi-select）

- **型**：Select（複数選択可にしたい場合は Multi-select）
- **説明**：実装内容の分類
- **選択肢**：
  - 🎨 **UI/デザイン**：画面・レイアウト関連
  - ⚙️ **機能実装**：新機能追加
  - 🐛 **バグ修正**：不具合対応
  - 🔧 **リファクタ**：コード整理・最適化
  - 📚 **ドキュメント**：ドキュメント更新
  - 🔐 **セキュリティ**：セキュリティ改善
- **入力方法**：手動

### 7. 影響範囲（Multi-select）

- **型**：Multi-select（複数選択）
- **説明**：実装の影響を受ける領域
- **選択肢**：
  - 🎭 **フロントエンド**：React・UI 関連
  - 🖥️ **バックエンド**：API・サーバ関連
  - 💾 **データベース**：DB スキーマ・クエリ関連
  - 🔐 **認証**：ログイン・権限関連
  - 📱 **モバイル**：モバイル対応
  - ☁️ **インフラ**：デプロイ・設定関連
- **入力方法**：手動

### 8. 実装者（Person）

- **型**：Person
- **説明**：チームメンバーの関連付け
- **メンバー**：
  - 吉沢（yoshizawa）
  - 佐藤（satoh）
  - 板垣（itagaki）
  - 高柳（takayanagi）
- **入力方法**：手動
- **補足**：GitHub ユーザ名との対応付けができます

### 9. レビュー担当者（Person）

- **型**：Person
- **説明**：PR レビュアー
- **入力方法**：手動
- **補足**：複数設定可能にする場合は People（複数選択）に変更

### 10. テスト状況（Select）

- **型**：Select
- **説明**：テスト実施状況
- **選択肢**：
  - ⚪ **未テスト**：テスト未実施
  - 🟠 **テスト中**：テスト実施中
  - ✅ **テスト済み**：テスト完了
  - ⚠️ **要再テスト**：追加テストが必要
- **デフォルト**：「未テスト」
- **入力方法**：手動

### 11. 備考（Text）

- **型**：Text / Rich Text
- **説明**：その他の補足情報
- **例**：「Supabase 統合が必要」「モバイル対応待ち」
- **入力方法**：手動

## オプションカラム

### 12. ファイル（File）

- **型**：File
- **説明**：関連ファイル・スクリーンショットのアップロード
- **用途**：デモ画像、ドキュメント、設計図など

### 13. 本番反映日（Date）

- **型**：Date
- **説明**：本番環境への反映日
- **入力方法**：手動

## ビュー設定推奨例

### ビュー 1: すべての実装（テーブル表示）

**設定**：
- タイプ：Table
- ソート：「実装日時」降順（最新が上）
- フィルター：なし

### ビュー 2: 進捗管理（ボード表示）

**設定**：
- タイプ：Board
- グループ化：「ステータス」
- 表示順序：未確認 → 確認中 → 完了

### ビュー 3: タイムライン（実装履歴）

**設定**：
- タイプ：Timeline
- 日付プロパティ：「実装日時」

### ビュー 4: 担当者別（ギャラリー表示）

**設定**：
- タイプ：Gallery
- グループ化：「実装者」

### ビュー 5: 未確認項目（フィルター）

**設定**：
- タイプ：Table
- フィルター：ステータス = 「未確認」
- ソート：実装日時（新順）

## データベース自動化

### ルール例 1: ステータス自動更新

トリガー：「実装日時」が入力された
アクション：「ステータス」を「未確認」に設定

### ルール例 2: テスト期限リマインダー

トリガー：「テスト状況」が「テスト中」から「テスト済み」に変更
アクション：「実装者」に @mention を送信

## ベストプラクティス

### 入力タイミング

1. **自動入力部分**（PRタイトル等）：PR マージ時に自動入力
2. **初期入力部分**（ステータス等）：当日中に入力
3. **詳細情報**（テスト状況等）：テスト完了後に入力
4. **本番反映情報**：デプロイ後に入力

### 命名規則

- カラム名：日本語で具体的に
- 選択肢：絵文字 + テキストで視認性向上
- タグ：簡潔かつ統一

### 定期確認

- **週次**：「未確認」「確認中」の項目を確認
- **月次**：実装量・機能分類の集計
- **四半期**：品質・テスト状況の分析
